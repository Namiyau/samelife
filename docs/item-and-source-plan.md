# SameLife V0.52 小项目与数据源设计

## 数据源分层

- SL官方：官方统计、官方票价、公共事业价格、企业公开价。适合作为基准。
- SL综合：官方价 + 平台样本 + 实际体验 + 常驻优惠券/配送费等。适合模拟真实生活。
- NUMBEO：众包生活成本数据。适合作为参考和对照，不建议作为唯一依据。

## 是否不同数据源使用不同小项？

建议尽量使用同一套类别和 item id。

原因：SameLife 比较的是“同一种生活方式”。如果数据源切换时小项也变化，用户很难判断差异来自城市还是来自问卷结构。

## 食品类建议按生活场景，而不是过细原材料

用户更容易回答“我每月下馆子几次”“我每月点外卖几次”，而不是回答“我每月吃多少克番茄”。

推荐食品小项：
- 自己做饭：普通一餐
- 便利店/轻食：一餐
- 普通外食：一餐
- 外卖：一餐
- 咖啡/奶茶：一杯
- 饮酒/酒吧：一次
- 零食/甜品：一次

原材料数据可以在后台作为“自己做饭”的计算依据，但不要直接让用户输入太多原材料频率。
